# 微信公众号

## 这个平台的本质

公众号是私域流量的载体。读者不是在"浏览网页"——他们在地铁上、在马桶上、在等人的间隙，用拇指快速滑动。

**关键洞察**：
- 阅读时间：平均 2-3 分钟，超过 5 分钟要有极强的内容
- 阅读姿势：单手握手机，拇指滑动
- 心理状态：寻找刺激、打发时间、获取谈资
- 信任来源：朋友转发 > 算法推荐

**什么内容会被转发**：
- 让转发者显得有品味/有见识/有趣
- 表达了转发者想说但说不出的观点
- 有实用价值（工具、方法、资源）
- 触发情绪（愤怒、共鸣、惊讶、感动）

## 写作心态

你不是在写文章，你是在和一个聪明但没耐心的朋友说话。

**想象读者**：
- 正在刷手机打发时间
- 随时准备划走
- 每隔 3 秒需要一个新刺激点
- 讨厌被教育，喜欢被启发

**你的目标**：
- 前 3 秒抓住注意力
- 每一段给一个留下来的理由
- 结尾给一个转发/收藏的理由

## 反模式：绝对不要做

**瞬间划走**：
- 开头是背景介绍："随着AI技术的发展..."
- 学术腔调："本文将探讨..."
- 长段落（手机上超过 4 行就是长段落）
- 没有视觉断点的文字墙

**看完不转发**：
- 没有金句/没有可截图的内容
- 观点太平庸/太安全
- 结尾软绵绵
- 没有行动召唤

**让读者觉得蠢**：
- 解释常识
- 居高临下的语气
- "你应该..."开头的说教

## 结构工程

**开头（前 50 字）**：
```
❌ "在当今数字化转型的大背景下，企业面临着..."
✅ "昨天，我删除了用了 5 年的 App。"
✅ "90% 的人对 X 的理解是错的。"
✅ "先说结论：别用 X。"
```

**正文节奏**：
```
短句。
短句。
稍长一点的句子来平衡节奏。

然后空一行。

**关键点加粗。**

- 列表
- 打破
- 单调

> 引用框创造视觉变化

再来一段短的。
```

**结尾（最后 100 字）**：
- 金句收尾（可以被截图的那种）
- 或者开放式问题引发讨论
- 或者行动召唤（关注、转发、评论）

## 标题的艺术

公众号标题是一门独立的学问。

**有效模式**：
- 数字具象化："月薪 3 万的人和月薪 3 千的人，差距在这 3 点"
- 对比冲突："为什么我劝你别学 Python"
- 悬念："他用 3 行代码，让公司损失了 200 万"
- 共鸣："那些 30 岁还在写代码的人，后来怎么样了"
- 反常识："996 是最低效的工作方式"

**失败模式**：
- 太学术："论 X 在 Y 中的应用研究"
- 太平淡："关于 X 的一些思考"
- 太长：超过 25 字就太长了
- 太抽象：没有具体的人/事/数字

## 专家级技巧

**钩子密度**：每 3-4 句话应该有一个"钩子"——让读者想继续看的理由。钩子可以是：
- 悬念："但事情没那么简单..."
- 转折："然而..."
- 预告："后面我会详细说"
- 利益："这个方法帮我省了 10 小时"

**情绪曲线**：好文章的情绪不是平的。
- 开头：制造张力
- 中间：提供价值 + 小高潮
- 结尾：情绪释放或升华

**可截图性**：写的时候想象这段话被截图发朋友圈会是什么效果。好的段落应该能独立成为一张有传播力的图片。

## 这才是专家水平

专家级的公众号文章：
- 开头 3 秒抓人，不需要"热身"
- 每一段都有存在的必要
- 能被一句话总结，但读完有更多收获
- 金句浑然天成，不是硬凑
- 结尾让人想转发或收藏
- 重新读一遍不觉得尴尬

**终极测试**：把你的文章发给一个不认识你的人。他会读完吗？会转发吗？为什么？
